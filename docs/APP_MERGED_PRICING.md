## 二、定价体系 v3.0（基于 API 成本的 1000x 加价率）

### 2.1 设计原则

1. **去掉终身买断**：改为「年付优惠」+「超大 Token 包」
2. **基于 API 成本定价**：确保 1000 倍加价率
3. **Token 包设计**：用户容易理解「能用多少次」
4. **年付享 17% 优惠**：锁定长期用户

### 2.2 API 成本基准

| 模型 | 输入成本 | 输出成本 | 说明 |
|------|---------|---------|------|
| GLM-4 Flash | ¥0.4/百万 tokens | ¥1.6/百万 tokens | 主力基础模型 |
| DeepSeek Chat | ¥1.8/百万 tokens | ¥7.2/百万 tokens | 主力高级模型 |
| DeepSeek R1 | ¥4/百万 tokens | ¥16/百万 tokens | 推理模型 |
| Groq Llama 3.3 | ¥1.2/百万 tokens | ¥1.2/百万 tokens | 极速模型 |
| Gemini 2.5 Flash | ¥0.5/百万 tokens | ¥2/百万 tokens | 多模态 |

**平均使用估算**：
- 轻度用户：每天 10 条消息 × 800 tokens = 240K tokens/月
- 中度用户：每天 50 条消息 × 800 tokens = 1.2M tokens/月
- 重度用户：每天 200 条消息 × 800 tokens = 4.8M tokens/月

## 五、技术架构（合并后）

### 5.1 Web + APP 统一架构

```
┌────────────────────────────────────────────────────────────┐
│                        用户端                              │
├────────────────────────────────────────────────────────────┤
│  Web (mcmamoo.com)     │     Android APP                   │
│  - 完整功能体验         │     - 移动端优化                  │
│  - 管理后台             │     - 离线缓存                    │
│  - 桌面端优先           │     - 推送通知                    │
├────────────────────────────────────────────────────────────┤
│                        API 网关                            │
│  (同一套后端，APP 和 Web 共享)                              │
├────────────────────────────────────────────────────────────┤
│  Supabase (数据库+认证) │ Qdrant (向量) │ AI 模型          │
└────────────────────────────────────────────────────────────┘
```

### 5.2 APP 技术选型

- **框架**：Capacitor (基于现有 Web 代码)
- **优势**：复用现有前端代码，降低开发成本
- **配置**：已创建 `capacitor.config.json`
