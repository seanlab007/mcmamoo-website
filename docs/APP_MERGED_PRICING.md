## 六、技术架构（合并后）

### 6.1 Web + APP 统一架构

```
┌────────────────────────────────────────────────────────────┐
│                        用户端                              │
├────────────────────────────────────────────────────────────┤
│  Web (mcmamoo.com)     │     Android APP                   │
│  - 完整功能体验         │     - 移动端优化                  │
│  - 管理后台             │     - 离线缓存                    │
│  - 桌面端优先           │     - 推送通知                    │
├────────────────────────────────────────────────────────────┤
│                        API 网关                            │
│  (同一套后端，APP 和 Web 共享)                              │
├────────────────────────────────────────────────────────────┤
│  Supabase (数据库+认证) │ Qdrant (向量) │ AI 模型          │
└────────────────────────────────────────────────────────────┘
```

### 6.2 APP 技术选型

- **框架**：React Native 或 Capacitor (基于现有 Web 代码)
- **优势**：复用现有前端代码，降低开发成本
- **推荐**：Capacitor (快速上线，后期可迁移 React Native)
